## 1. Где лежит проект и как с ним работать

- **Репозиторий:** `arniByte/CasasFortunae` (GitHub). Доступ — через GitHub MCP
  (`mcp__github__*`), а также обычный `git` (push работает, установлен Claude GitHub App).
- **Основная ветка:** `main`. Рабочая ветка для разработки: `claude/vibrant-bohr-z6fpmp`.
  После мержа PR ветку сбрасываем на свежий `main` (`git checkout -B claude/vibrant-bohr-z6fpmp origin/main`).
- **Workflow:** разработка на ветке → коммит (подписанный, `-S`) → push `-u origin <branch>`
  → создать **draft PR** → снять с draft → squash-merge в main → синхронизировать локальный main.
- **Коммиты подписаны** SSH-ключом (gpgsig). В конце commit message:
  `Co-Authored-By: Claude Sonnet 4.6 <noreply@anthropic.com>` + `Claude-Session: ...`.
- **CI в репо нет** (0 workflows) — проверки гоняем локально вручную.
- **Зависимости** ставятся SessionStart-хуком (`.claude/hooks/session-start.sh`)
  в web-сессиях: `pip install --ignore-installed blinker -r requirements.txt`.

### Песочница — ограничения
- **Нет интернета к Met API (403)** — аватары/фоны падают на SVG/градиентные заглушки.
  На машине пользователя грузятся. В тестах это НЕ баг.
- **Playwright недоступен** (нет браузеров) — UI-тесты гоняем альтернативно:
  Flask `test_client().get('/')` (рендер 200 + наличие нужных строк) и прямые
  юнит-проверки `GameState` из Python.

### ⏳ ОТЛОЖЕННАЯ ЗАДАЧА — визуал (ждёт пользователя)
Пользователь хочет ручную отрисовку как на ч/б референсе (геральдический орёл на
знамени с кистями + конный рыцарь). Решение: **векторизовать реальный референс**
(не ИИ-SVG). Пайплайн готов:
- `potrace` 1.16 (apt) + `vtracer` (pip) + `cairosvg` (pip) — установлены в песочнице
  (НО переустанавливать в новой сессии, окружение эфемерно).
- Скрипт `/tmp/art/trace.py` (НЕ в репо): PIL preprocess → potrace → перекраска
  в чернила `#241910` → PNG-превью на пергаменте.
- **Блокер:** нужен файл-референс в репо. Просили загрузить в `art/refs/reference.png`
  (папка `art/refs/` создана, пустая). Как появится — `git pull`, прогнать трассировку,
  показать результат, затем вставить в: гербы Домов (`HOUSE_CREST`), экран входа
  (`#entry`), иллюстрации карт (`CARD_ART`/`cardArtSVG`), баннер хода.
