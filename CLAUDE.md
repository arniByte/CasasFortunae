## 1. Что это

**Casas Fortunae** — браузерная карточная дуэль на двоих в эстетике итальянского
Возрождения. Компьютер (ноутбук) поднимает локальный сервер; два телефона заходят
по Wi-Fi в браузере на `http://<ip>:5000` — без установки приложений. Сервер на
Python (Flask + flask-sock / WebSocket), клиент — один самодостаточный HTML-файл.

Два знатных Дома сходятся под вращающимся Колесом Фортуны. Цель — снизить
**Престиж** соперника с 30 до 0.

Эталонный визуальный референс — игра «Death of the Reprobate» (коллажи из полотен
Северного Возрождения) и геральдические рисунки от руки (грифоны, гербы, лозы).

## 2. Быстрый старт

```bash
# зависимости
pip install flask flask-sock qrcode   # qrcode опционален (QR-код при старте)

# запуск (из папки проекта ИЛИ откуда угодно — пути привязаны к server.py)
python server.py
# печатает LAN-адрес + QR, биндится на 0.0.0.0:5000
```

Открыть на двух телефонах напечатанный адрес → ввести имя → вкладка «Быстрая игра»
(общая комната `MAIN`) → выбрать Дом и портрет → «Готов». Когда оба готовы — партия
стартует.

### Тестирование (Playwright, два контекста)
Тесты гоняются через два headless-браузера (390×844). Для звука нужен флаг
автозапуска, иначе аудио не стартует в headless:
```python
b = pw.chromium.launch(args=['--autoplay-policy=no-user-gesture-required'])
```
Запуск сервера из любой папки: `PYTHONPATH=<projdir> python3 server.py`.
Чтобы обучение не открывалось в тесте: `page.evaluate("localStorage.setItem('cf_tut','1')")`.
**Песочница не может ходить в интернет к Met API (403)** — аватары и фоны падают на
встроенные SVG/градиентные заглушки; на машине пользователя они грузятся.
Проверять: 0 `pageerror`, баланс тегов (`<style>` 3/3, `<script>` 1/1),
`python3 -c "import ast; ast.parse(open('server.py').read())"`.

### Симулятор баланса — `simulate.py` (Направление 1 плана — построен)
Headless-прогон реального `GameState` (без браузера и сети) с 4 ботами-стратегиями
(aggressor / economist / random / adaptive). Импортирует движок прямо из `server.py`,
поэтому всегда тестирует НАСТОЯЩУЮ логику. Считает: винрейт Дом×Дом (матрица + агрегат),
матрицу стратегий, длину партий, % fatigue/перетасовки, частоту Долга и реликвий,
преимущество первого хода. Детерминирован по `--seed` (подменяет `secrets` на seeded
`random` только в симуляторе). Отчёт пишется в `SIM_REPORT.md`.
```bash
python3 simulate.py                        # дефолт: 500 партий/пара Домов (~20 с)
python3 simulate.py --games 800 --seed 7   # точнее / другой сид
```
**Правило:** гонять ДО и ПОСЛЕ любой правки баланса — показывать разницу числами,
а не «на глаз». Статус всех фич — в `FEATURES.csv` (каноническая таблица); рабочий
контекст и история — в `HANDOFF.md` (мастер-файл для передачи между сессиями).
*Ограничения:* боты простые; Долг Фортуне ими почти не берётся (слепое пятно симуляции);
Медичи — Дом ценности/темпа, простые боты пилотируют его слабо → его винрейт занижен
(истинная сила между «бот не умеет» и «реально слаб»). Веса оценки DoT (`POISON_W`
и т.п.) — эвристика, влияет на матчап Борджиа↔Сфорца, к ним стоит относиться осторожно.

## 3. Технологический стек

- **Сервер:** Python 3, Flask, flask-sock (WebSocket без async/eventlet), threading.
- **Клиент:** один файл `templates/game.html` — HTML + CSS + ванильный JS, без сборки,
  без фреймворков, без внешних зависимостей (шрифты Google Fonts по CDN).
- **Аудио:** HTML5 `<audio>` (музыка) + Web Audio API (синтез звуков боя, без файлов).
- **Графика:** инлайновый SVG, рукописный «чернильный» вид через SVG-фильтры
  (feTurbulence + feDisplacementMap).
- **Внешние данные:** Met Museum Open Access API (портреты-аватары и картины-фоны),
  запрашивается **на стороне сервера** (обход CORS), кешируется, с graceful-фолбэком.

## 5. Архитектура

**Сервер — единственный источник правды.** Вся логика на Python; клиент только
рисует состояние. После каждого изменения сервер шлёт полное состояние; рука
соперника вырезается из его вида.

- Flask отдаёт один HTML; пути к `static`/`templates` **абсолютные**, привязаны к
  `BASE_DIR = os.path.dirname(os.path.abspath(__file__))` — НЕ к рабочей директории.
  Это критично (см. §12, баг с музыкой 404).
- WebSocket на `/ws` (flask-sock, threaded). Комнаты в памяти по 4-буквенному коду
  (по умолчанию `MAIN`). `threading.Lock` (`STATE_LOCK`) защищает состояние.
- Идентификация игрока — UUID-токен в `localStorage`; авто-реконнект с
  экспоненциальной задержкой.
- Вращение Колеса серверо-авторитетно: сервер шлёт финальный угол, оба клиента
  анимируют одинаково.
- LAN: при старте печатает IP + QR, биндится на `0.0.0.0:5000`.

### Поток экранов
`entry` (ввод имени/комнаты) → `select` (выбор Дома + портрета, готовность) →
`board` (игра) → `over` (итог). Состояние строится лениво на `start`, когда оба
игрока готовы и выбрали Дом.

### WebSocket-сообщения (клиент → сервер)
`hello` · `create_room` · `choose {house, avatar, ready}` · `start` ·
`play_card {uid}` · `take_debt` · `respin` · `end_turn` · `use_relic` · `new_game`.

Сервер → клиент: `lobby` (с `both_ready`), полное `state` (через `view_for`),
`error`. Подробная схема — в `architecture.md` скилла.

## 6. Игровая механика (v2/v3)

- **Цель / победа:** снизить Престиж (♛, здоровье) соперника с **30** до 0
  (`START_PRESTIGE = 30`).
- **Рука:** старт `START_HAND = 4`.
- **Колесо Фортуны:** в начале хода благословляет 1 из 4 стихий. Благословлённые
  карты стоят на 1 **дешевле** и эффект на 1 **сильнее** (Дом Эсте: +2 вместо +1).
- **Флорины (ресурс):** в начале хода `+= min(turns_taken, FLORIN_CAP=10)`.
  Накапливаются между ходами (без авто-сброса), но обрезаются потолком
  `MAX_FLORINS = 15` (см. §13) — копить про запас можно, бесконечно нет.
- **Броня:** не убывает сама по себе, но обрезается потолком `MAX_ARMOR = 20` (§13).
- **Стрик Фортуны:** одна стихия благословлена два хода подряд → +1 флорин.
- **Долг Фортуне (`take_debt`):** +3 флорина сейчас, но соперник управляет твоим
  следующим вращением Колеса.
- **Стихии:** `fire` (Ignis), `water` (Aqua), `earth` (Terra), `air` (Aer).
  Цвета (тёплая палитра): fire `#B23A22`, water `#2E6A62` (бирюза, НЕ синий),
  earth `#5A6A38`, air `#C99A2E`. Есть `neutral`.

### Дома (фракции) — `HOUSES` в server.py
| Дом | Цвет | Пассивка | Реликвия |
|-----|------|----------|----------|
| **Медичи** | `#B8901F` | Покровительство: +2 флорина/ход | Золотая книга — раз/ход: 3 флорина → карта |
| **Борджиа** | `#6B2A40` | Яд в крови: первый урон за ход даёт +1 яд | Флакон Кантареллы — раз/игра: удвоить весь яд врага |
| **Сфорца** | `#524A38` | Дисциплина: +1 брони/ход | Знамя капитана — раз/ход: 4 брони → 4 урона |
| **Эсте** | `#2E6A62` | Муза: благословлённые карты +2 и +1 престиж | Астролябия — раз/ход: снять 1 яд + флорин / подсмотреть Колесо |
| **Савонарола** | `#5A3320` | Праведный гнев: пока отстаёшь по престижу — урон +2 | Власяница — раз/ход: −2 престижа → 4 урона в обход брони |

Каждый Дом = тематическая колода ~26 карт (`build_house_deck`). **5 Домов.**
Савонарола (Дом проповедников) — ниша **жертвы/самопожертвования**: платит престижем
и картами за испепеляющие всплески; пассивка превращает отставание в силу. См. §13f.

### Статусы — `STATUS_GLYPH` / `STATUS_RU`
| id | глиф | рус | поведение |
|----|------|-----|-----------|
| `poison` | ☠ | Яд | тикает в начале хода, **игнорирует броню**, −1 стак/ход |
| `bleed` | 🩸 | Кровотечение | теряешь 1/сыгранную карту, игнорирует броню |
| `burn` | 🔥 | Горение | тикает, потом стак уполовинивается |
| `weaken` | 🥀 | Ослабление | −1 к урону |
| `inspiration` | ⚡ | Вдохновение | следующая карта бесплатна |
| `blessing` | ✨ | Благословение | поглощает следующий негативный статус ИЛИ урон |
| `fortified` | 🛡 | Укрепление | броня непробиваема |

Дизайн-смысл: DoT (яд/кровь/горение) **игнорирует броню** → контра против туртла;
Благословение — универсальный «снять/щит».

### Карты
- `CATALOGUE` в server.py: **50 карт**, фабрика `C(cid, name, element, cost, text,
  kind, value, house, rarity, extra)`.
- **39 типов (kind):** damage, damage_status, damage_break, damage_pierce, charge,
  surge, thorns, heal, heal_draw, armor, armor_status, armor_heal, armor_blessing,
  armor_damage, draw, draw_damage, draw_discount, status_draw, status_self,
  cleanse_self, gain_florin, florin_draw, florin_heal, invest, tax_enemy, spend_damage,
  poison_double, poison_skip, peek_draw, peek_respin, return_card, codex, harmonic,
  immolate, tithe, penance, zeal, flagellant, martyr.
  (последние 6 — Савонарола; `spend_damage` — финишер Медичи «Военный подряд»:
  тратит до 8 флоринов → столько же урона, win-condition для экономики.)
- `kind` рулит движком эффектов в `play_card`. `rarity == "artifact"` — редкие
  правила-ломающие карты.

## 7. Карта кода: server.py (~1011 строк)

- **Константы/данные:** `START_PRESTIGE=30`, `START_HAND=4`, `FLORIN_CAP=10`,
  `HOUSES`, `ELEMENT_RU`, `STATUS_GLYPH/RU`, `CATALOGUE` (через `C()`),
  `build_house_deck(house_id)`.
- **`PlayerState`** (seat, name, house, avatar): `statuses` dict, `prestige`,
  `armor`, `florins`, `deck/hand/discard`, `add_status`, `has_status`, `draw`.
- **`GameState`**: `spin_wheel`, `start_turn` (тики статусов + пассивки + экономика),
  `_tick_statuses`, `_deal` (урон с pierce/break_armor + поглощение благословением +
  яд Борджиа), `_apply_status`, `_cost`, `_value`, `_est_damage`/`_lethal_threat`
  (оценка угрозы для бейджа «под угрозой», §13c), `play_card` (движок эффектов по
  kind), `use_relic`, `take_debt`, `respin`, `end_turn`, `_check_over`,
  **`view_for(seat)`** (сериализация состояния; рука соперника вырезается; поле
  `under_threat` на каждого игрока).
- **Внешние API:** `_met_fetch`, `load_avatars` + `/api/avatars`;
  `MUSIC_TRACKS` + `/api/music`; **`/static/music/<filename>` (явный маршрут,
  send_from_directory, mimetype audio/mpeg)** + `music_status()`;
  `load_backgrounds` (картины Met) + `/api/backgrounds`; `/api/houses`.
- **Сеть/комнаты:** `Room` (broadcast), `get_or_create_room`, `new_code`,
  `_safe_send`, `ws_handler` (обработка всех сообщений), `lan_ip`, `print_banner`
  (печатает адрес/QR + предупреждение, найдена ли музыка), `_prefetch` (аватары+фоны
  в фоне при старте).

## 8. Карта кода: templates/game.html (~1132 строки)

Структура: `<head>` стили → `<body>`: глобальные SVG-фильтры (`#inkG`,`#inkS`) →
`#entry` (кодекс) → `#stage` (перестраивается под select/board/over) →
постоянные элементы вне stage (переживают перестройку DOM): `#bgGallery`,
`<audio id="player">`, `#musicDock`, `#tutorial` → три `<style>`-блока → один `<script>`.

### Контроллеры (JS-объекты)
- **`Music`** (стр. ~448): постоянный `<audio>` вне `#stage` (НЕ прерывается при смене
  экранов). `load/renderPicker/renderDock/pick/togglePlay/next/kickstart/init`.
  Автозапуск — после первого жеста (`_firstGesture`). Селектор треков на `#entry`,
  док справа сверху (на доске — компактный `.mini`). Обработчик `error` показывает
  «Трек не найден».
- **`Backgrounds`** (~526): два слоя `.bg-layer`, кроссфейд + Ken Burns. Берёт картины
  Met (`/api/backgrounds`); если пусто — встроенные `BUILTIN_BG` градиентные сцены.
  `dim(on)` притушает фон в игре.
- **`SFX`** (~572): синтез звуков через Web Audio API (без файлов). `_at/_glide/_noise`
  → `_s_hit/_s_heal/_s_armor/_s_coin/_s_spin/_s_poison/_s_bless/_s_win/_s_lose/
  _s_card/_s_tick`. Триггерятся по диффу состояния в `updateBoard` (урон→hit,
  лечение→heal, броня→armor, статусы→poison/bless, Колесо→spin, финал→win/lose) +
  тапы карт. Кнопка 🔔 (`toggle`) в доке.
- **`Tutorial`** (~649): 6 шагов кор-механик, инлайновые SVG-эмблемы. Открывается с
  меню и авто при первом визите (`maybeAutoOpen`, флаг `localStorage 'cf_tut'`).

### Рисованная графика (рукописный стиль)
- `elemSigInner(el,ink)` / `elemSig(...)` — чернильные сигилы стихий (на Колесе и
  полоске карты).
- `CARD_ART` (sword/shield/chalice/coins/skull/scroll/flame/star/vial/mountain/swirl)
  + `cardArtKey(c)` + `cardArtSVG(c)` — гравюрные иллюстрации карт по kind/element/rarity.
- `wheelSVG()` — Колесо (сепийная сердцевина, золотой ободок с заклёпками, сигилы).
- `laurelSVG()` — лавровые ветви у баннера хода (анимация `laurelSway`).
- `flourishSVG()` — растительные завитки по углам кодекса входа.
- Логотип-герб с грифоном — инлайн в `#entry` (фильтр `#inkLogo`).
- Запасные портреты: `AV_PALETTES` + `svgAvatar(idx)` (когда Met недоступен).

### Игровой цикл (клиент)
`connect/send/handle` → `showSelect/buildSelect/renderHouses/renderAvatars` →
`renderGame/buildBoard/updateBoard` (диффы для вспышек и звуков) →
`renderHand/tapCard` (двойной тап: выбрать → сыграть) → `renderActions` →
`showOver` → `resetAll`.

## 9. Визуальный стиль (КАНОН — соблюдать)

**Палитра — тёплая, концентрированная. СИНЕГО НЕТ.** CSS-переменные в `:root`:
- Пергамент: `--gesso #F4ECD6` (молочный), `--vellum #E9DCBE` (беж), темнее —
  `--vellum-deep/-dark`, `--parch-shadow`.
- Золото: `--gold-leaf #B8901F`, `--gold-bright #D9AE3A`, `--gold-pale #E8CE86`,
  `--gold-shadow #7E5E16`, `--gold-deep`.
- Коричневый (чернила/рамки/текст): `--brown #4A3120`, `--ink-black #241910`.
- Тёмно-зелёный: `--green #46552C`, `--green-earth #5A6A38`.
- Бордовый: `--burgundy #7E2230`, `--madder #8A2A2A`, `--vermilion #B23A22`.
- Вода = бирюза: `--teal #2E6A62` (НЕ ярко-синий).
- Тёмная подложка: `--night #241B10` / `--night-deep #160F08` (сепия, не индиго).
- Легаси-алиасы (`--ultramarine` и т.п.) **перенаправлены** на тёплые цвета — старые
  правила резолвятся в бордовый/сепию, а не в синий. Новый синий не вводить.

**Рисованный «иллюминированный» вид (v4):**
- Глобальные фильтры `#inkG`/`#inkS` (feTurbulence + feDisplacementMap) дают дрожащую
  «чернильную» кромку. Любой новый SVG — со `stroke-linejoin/linecap="round"` и
  `filter="url(#inkS)"`.
- Чернила: `#241910` на пергаменте, `#F4ECD6` на тёмном. **Никогда не чистый чёрный.**
- Карты: иллюстрация по смыслу + двойная рукописная рамка (`.card-art::before`) +
  пергаментный шум (`.card-art::after`).
- Шрифты: Cinzel (капитель/заголовки), EB/Cormorant Garamond (текст). Кириллический
  блэклеттер НЕ использовать (его почти нет, читается плохо).

**Контраст (выучено больно):** на светлом пергаменте основной текст НЕ делать золотым/
бледным — пропадает. Заголовки — бордовые/ультрамарин-алиас (тёмный). Золото — только
для рамок, медальонов, гербов, разделителей. На тёмном фоне-картине золотой текст ок.

## 10. Как расширять

- **Новая карта:** добавь в `CATALOGUE` через `C(...)` с корректным `kind`; убедись,
  что `play_card` обрабатывает этот `kind`; добавь в нужную колоду в
  `build_house_deck`. Клиент сам подберёт иллюстрацию по `cardArtKey` (или расширь его).
- **Новый статус:** добавь в `STATUS_GLYPH/RU`, тик в `_tick_statuses`/`start_turn`,
  дай ему контрплей (обычно Благословение/очищение), стиль бейджа `.sb-<status>` в
  клиенте, и классификацию в дифф звуков (NEG_STATUS/POS_STATUS).
- **Новый Дом:** добавь в `HOUSES` (пассивка+реликвия+цвет из палитры), колоду в
  `build_house_deck`, герб `HOUSE_CREST` в клиенте, пассивку в `start_turn`, реликвию
  в `use_relic`.
- **Новая иллюстрация/сигил:** добавь ключ в `CARD_ART` (инлайн SVG-пути в чернильном
  стиле) и маппинг в `cardArtKey`.

## 12. Подводные камни (важно!)

1. **Музыка 404 (был баг):** Flask искал `static/` относительно рабочей директории.
   Лечилось: абсолютные `static_folder/template_folder` от `BASE_DIR` + явный маршрут

2. **Autoplay:** браузеры не пускают звук без жеста. Музыка и Web Audio стартуют в
   `_firstGesture` (первый `pointerdown`/`keydown`). На реальном телефоне тап = жест.

3. **Met API в песочнице = 403** (нет интернета). Аватары/фоны падают на заглушки.
   На машине пользователя — работают. Не считать это багом в тестах.
4. **Доставка:** отдавать **ZIP** (`shutil.make_archive`), а не отдельные файлы —
   иначе пользователь теряет структуру папок (и снова ловит музыку-404).
5. **Постоянные элементы** (`<audio>`, `#bgGallery`, `#musicDock`, `#tutorial`) ДОЛЖНЫ
   быть вне `#stage` — иначе перестройка DOM их убивает (музыка прервётся).
6. **localStorage** работает (это обычный сервер у пользователя, не claude.ai-артефакт).
7. Баланс тегов после правок CSS: `<style>` 3/3, `<script>` 1/1 — проверять.

## 13. Баланс-патч (выполнен)

Проведён анализ всех 43 карт каталога по соотношению эффект/стоимость (value/cost) —
выявлены и устранены конкретные перекосы:

1. **Потолки ресурсов (новое):** `MAX_FLORINS = 15`, `MAX_ARMOR = 20` (константы рядом
   с `FLORIN_CAP`). Метод `GameState._clamp_resources()` обрезает оба значения у обоих
   игроков; вызывается в конце `start_turn`, `play_card`, `use_relic`, `take_debt`.
   **Почему:** флорины и броня раньше росли бесконечно без сброса — это создавало
   риск деградации в стратегию «копи 20+ ходов → взорви всё за один ход» и
   «нерушимая стена Сфорца» (его пассивка +2 брони/ход без декея). Потолки сохраняют
   возможность копить про запас на 2-3 хода, но убирают бесконечный снежный ком.
   Протестировано: 60 ходов чистой экономики без розыгрыша карт — оба значения
   корректно упираются в потолок и не переливаются.
2. **Кривая стоимости карт выровнена:**
   - `ward` («Каменный оберег»): cost 1→2 (был самым эффективным во всей игре,
     3.0 — теперь 1.5, в одном ряду с torch/apoth/riposte).
   - `pike` («Стена пик», Сфорца): cost 2→3 (было 2.0+бонус-статус — теперь 1.33,
     на уровне charge/surge).
   - `baptism` («Крещение»): heal 4→5 (было слабовато для cost 5, ratio 0.8→1.0).
   - После правок верхушка кривой (`rampart` 2.0 на cost 3) стала вознаграждать
     осмысленное вложение в более дорогую карту, а не спам дешёвки.
3. **Исправлен баг карты `surge`** («Стихийный всплеск», Эсте): условие
   «если любая стихия благословлена» было **всегда true** (Колесо всегда благословляет
   ровно одну стихию каждый ход) — карта фактически била на фиксированные 4 урона
   без всякого условия, при ложной формулировке текста. Перепривязано к реальному
   стрику Колеса (`self.last_blessed == self.blessed`): 4 урона обычно, **6** если та
   же стихия благословлена второй ход подряд. Это даёт Эсте осмысленную причину
   следить за повторами Колеса (синергия с уже существующим бонусом стрика +1 флорин).

**Обучение обновлено:** шаг про флорины больше не говорит «сгорают» (это никогда не
было правдой — копились без сброса); теперь говорит про потолок 15.

## 13b. Баланс-патч v2 (Направление 2) — выполнен

Построен headless-симулятор (`simulate.py`, §2) и на его основе проведён структурный
баланс Домов. До патча: Сфорца 79.5%, Борджиа 67.3%, Эсте 39.4%, Медичи 13.8%.

**Изменения:**

1. **Сфорца — ключевой нёрф «стены»:**
   - Пассивка «Дисциплина»: +2→**+1** броня/ход (главная причина непробиваемости).
   - `pike` («Стена пик»): cost 3→**4** (Укрепление теперь дороже).
   - `riposte` («Стойка рипоста»): 3→**2** брони (немного меньше накопления).
   - `cannon_r` («Королевская пушка»): урон ограничен `min(броня, 10)` — нет сноса
     с 20 брони за один ход.

2. **Медичи — экономический разгон:**
   - Пассивка «Покровительство»: +1→**+2** флорина/ход.
   - `banco` («Вклад банка»): вложить 3→**2**, вернуть 5→**6** (лучший ROI).
   - `commission` («Заказ мастеру»): добавлен **+1 карта** (связка экономика→темп).
   - `bribe` («Подкуп стражи»): добавлен **2 урона** + налог снижен с 3 до 2
     (давление вместо чистого задержания).

3. **Борджиа — точечное ускорение яда (контр Сфорца):**
   - `apple` («Яблоко Борджиа»): cost 4→**3** (быстрее к контр-артефакту).

4. **Эсте — стабилизация стекла:**
   - `surge` («Стихийный всплеск»): cost 3→**2**, добавлен **pierce=2** (2 урона
     всегда обходят броню; стрик даёт 6+2 пробоя). Это позволяет Эсте давить
     сквозь стену Сфорца.
   - `harmonic` («Гармония сфер»): cost 5→**4** (доступнее ключевая карта).
   - **Муза (пассивка):** при разыгрывании карты благословлённой стихии
     восстанавливает **1 престиж** (частичная компенсация DoT Борджиа).
   - **Астролябия (реликвия):** снимает 1 стак яда + **+1 флорин** за ход;
     если яда нет — подсматривает следующую стихию.

**Результаты (800 партий/пара, seed=1):**

| Дом | До | После |
|-----|----|-------|
| Сфорца | 79.5% ⚠️ | **70.1%** ⚠️ |
| Борджиа | 67.3% ⚠️ | **66.2%** ⚠️ |
| Эсте | 39.4% ⚠️ | **50.7%** ✅ |
| Медичи | 13.8% | 13.0% (бот-ограничение) |

Борджиа→Сфорца контр: **32%→60%** ✅ (задуманный контр теперь работает).
Борджиа→Эсте: **88%→50%** ✅ (матчап выровнен).
Сфорца→Эсте остаётся 89% — «стена» vs «стекло», дизайн-обоснованный хард-контр.

## 13c. Индикатор угрозы (Направление 3) — выполнен

Добавлено тактическое напряжение через видимый сигнал «под угрозой» — без изменения
баланса (чисто информационная механика, симулятор подтвердил идентичные винрейты
до/после).

**Сервер (`server.py`):**
- `GameState._est_damage(src, tgt, c)` — чистая оценка урона карты по престижу
  без мутаций; зеркалит `_deal` (учитывает броню, pierce, break_armor, благословение,
  fortified, weaken, charge-бонус, surge-стрик, cap пушки).
- `GameState._lethal_threat(atk_seat, def_seat)` — `True`, если защитник рискует
  погибнуть к началу своего следующего хода: (а) собственный DoT (яд+горение)
  уже ≥ престижа, ИЛИ (б) у атакующего есть карта по карману (бюджет = текущие
  флорины + доход следующего хода + пассивки + возврат вклада), закрывающая одним
  ударом. Возвращает **только булев флаг** — карты соперника не раскрываются.
- `view_for`: каждому игроку добавлено поле `under_threat` (считается для обоих).

**Клиент (`templates/game.html`):**
- `threatHTML(p, isYou)` — бейдж под полосой игрока: «☠ Вы под угрозой» (красный,
  пульсирует) себе / «⚔ Момент удара» (золотой) на сопернике, когда его можно добить.
- CSS `.threat-row/.threat-danger/.threat-kill/.threat-pulse` + кейфреймы
  `threatThrob` (свечение) и `threatPulse` (масштаб глифа). В каноне палитры
  (madder/gold, без синего).
- Звук: при ПЕРВОМ появлении угрозы на тебе — тихий `tick` (диф `prevThreat`).

**Дизайн-смысл:** превращает «посчитай арифметику» в «реши — защищаться или бить
первым». Сигнал честный (показывает реальную возможность добивания), но не палит
конкретные карты — сохраняет блеф.

## 13d. Лимит карт за ход (Направление 3) — выполнен

Добавлен потолок **`CARDS_PER_TURN = 3`** (константа рядом с `MAX_ARMOR`): за один ход
нельзя сыграть больше 3 карт. Превращает ход из «разыграй всё по карману» в выбор
порядка и приоритета.

**Сервер:** `PlayerState.cards_played` (сброс в `start_turn`), проверка в начале
`play_card` (возврат строки-ошибки при достижении лимита → бот в симуляторе сам
завершает ход), инкремент при успешном розыгрыше. `view_for`: top-level
`cards_per_turn` + per-player `cards_played`.

**Клиент:** чип «N/3 карт за ход» в `action-bar` (`.act.plays`, золотой счётчик,
краснеет при 0); карты в руке недоступны при достижении лимита (`atLimit` в
`renderHand`); точное сообщение в `tapCard`; обновлён шаг обучения.

**Баланс (симулятор, 500 партий/пара):** на уровне Домов почти без изменений
(Сфорца 70→71%, Эсте 50.7→50.4% — в пределах шума) — лимит не ломает баланс.
Побочно вырос fatigue (54→60%) и перетасовка (71→79%): ходы чуть сдержаннее,
колода прокручивается чаще. Для живых игроков лимит сильнее бьёт по burst/combo-ходам
(codex, экономический разгон Медичи) — это и есть искомое тактическое напряжение.

## 13e. Сострадание Фортуны — мягкий комебэк (Направление 4) — выполнено

**Данные сначала:** замер снежного кома (2000 партий, adaptive) показал, что игра
**уже почти не снежит** — большое лидерство (≥8 престижа на 12-м ходу) конвертируется
лишь в **56%** побед, малое (≥3) — 49% (монетка). По плану сильный комебэк нужен,
только если лидер побеждает «почти всегда» — здесь это не так. Поэтому выбран
**самый мягкий** вариант (план 4b), без выдачи ресурсов даром.

**Механика:** `COMPASSION_THRESHOLD = 10`. Когда престиж активного игрока ≤ 10,
его вращение Колеса в начале хода **не случайно** — благословляется стихия, которой
у него больше всего карт в руке (`_compassion_element`, ничья — случайно среди
лидеров по `rng`). Это даёт отстающему скидки и усиление на его же карты (а Эсте —
ещё и лечение Музы). Долг Фортуны имеет приоритет (если соперник управляет твоим
вращением — Сострадание не срабатывает, см. `start_turn`).

**Сервер:** `spin_wheel(force=None)` (рефактор), `_compassion_element(p)`,
флаг `self.compassion` (сброс/установка в `start_turn`), в `view_for` —
`compassion` (true только активному игроку, получившему помощь).

**Клиент:** тег Колеса меняет текст на «Сострадание Фортуны: <стихия>» и подсвечивается
золотым свечением (`.bless-tag.compassion`, кейфрейм `compassionGlow`); тихий перезвон
`bless` при срабатывании (диф `prevCompassion`); добавлен 7-й шаг обучения.

**Эффект (симулятор):** конверсия большого лидерства 56→55% (чуть прощает),
длина партии 38→38.8 ходов, срабатывает в **33% партий** (ощутимо, но не доминирует).
Баланс Домов не сдвинут (Сфорца 71.2%, Эсте 49.9% — в пределах шума). Ровно тот
мягкий нудж, что задумывался — драма и тематика без переусиления отстающего.

## 13f. 5-й Дом — Савонарола (Направление 5) — выполнено

**Дом проповедников** (id `savonarola`, цвет `#5A3320` — обожжённая сиена). Историческая
основа — Джироламо Савонарола и «Костёр тщеславия» (Флоренция, 1497). Ниша —
**жертва/самопожертвование** (свободна, не дублирует экономику/DoT/броню/стихии).

**Идентичность:** платишь СВОИМ (престиж, карты) ради испепеляющих всплесков; чем
хуже дела, тем сильнее бьёшь — встроенный комебэк-аркетип.

- **Пассивка «Праведный гнев»:** пока твой престиж ниже вражеского, твои карты урона
  бьют на **+2** (реализовано в `_deal`: `source.house=="savonarola" and
  source.prestige < target.prestige` → +2 к amount или pierce; не применяется к
  самоурону через `source is not target`).
- **Реликвия «Власяница»:** раз/ход — −2 престижа, 4 урона в обход брони (`use_relic`).
- **Колода (6 карт Дома + 19 общих):** `immolate` (−2 себе → 6 пирса), `tithe`
  (−2 себе → 4 флорина + карта), `penance` (сбросить карту → +4 престижа + Благословение),
  `zeal` (3 урона +1 за каждые 4 недостающего престижа — растёт в беде), `flagellant`
  (−2 себе → Кровотечение 2 врагу + карта), `martyr` (артефакт: урон = половине твоего
  престижа в обход брони, теряешь столько же).

**Новые kind в `play_card`:** immolate, tithe, penance, zeal, flagellant, martyr.
Пирс-карты (immolate/martyr) бьют через `_deal(opp, 0, pierce=d)` — броня не снимается.
`penance` сбрасывает самую дорогую другую карту руки.

**Клиент:** герб `HOUSE_CREST.savonarola` (костёр + крест проповедника, ink-стиль);
`cardArtKey` — flagellant→skull, penance→chalice, tithe→coins, immolate/zeal→flame
(через fire-фолбэк), martyr→star (артефакт). Дом авто-появляется в выборе (из
`/api/houses`).

**Симулятор обучен (важно для оценки):** `DAMAGE_KINDS`/`PIERCE_KINDS` расширены;
`est_damage` считает immolate/zeal/martyr + бонус гнева; `self_damage()` вычитается
в `card_pressure` и **запрещает суицидальный розыгрыш** (`-999`, и в `_lethal_card`);
`_relic_ready` + choosers знают про Власяницу.

**Баланс (600 партий/пара, seed 7):** Савонарола **50.7%**, зеркало 49% (без перекоса).
Матчапы: vs Медичи 71%, vs Эсте 46%, vs Сфорца 36%, vs Борджиа 36% (жертва тяжела
против DoT/брони, но пассивка гнева компенсирует). Дебютная итерация (37%→50.7%)
выровнена баффами: пассивка +1→+2, immolate self 3→2, zeal база 2→3/шкала 5→4.
Побочно Сфорца/Борджиа просели к коридору (новый соперник давит лидера). Медичи 16%
без изменений (известное слепое пятно ботов).

## 13g. Аудит фич, win-condition Медичи и UX-фиксы — выполнено

Полный проход по всем фичам приложения с user story и ожидаемым поведением —
зафиксировано в `FEATURES.csv` (75 строк, каноническая таблица статусов).

**Win-condition Медичи (`condotta` «Военный подряд»):** новый kind `spend_damage` —
тратишь до 8 флоринов, наносишь столько же урона (cost 1, заменил дублирующий
`merchant` в колоде Медичи). Причина: у Медичи раньше НЕ было способа конвертировать
экономику в урон (единственный урон — bribe 2), поэтому богатство упиралось в потолок
15 и не превращалось в победу — Дом проигрывал Сфорца/Борджиа почти 0%. Это была не
только ботовая, но и человеческая проблема. Симулятор: Медичи **15.7%→22%** (боты
теперь пилотируют конверсию; истинная сила выше — тонкий темп им всё ещё не даётся).
Сфорца/Борджиа (68/62%) не нёрфил: их «высота» — в основном разгром бото-слабого
Медичи и дизайн-контр Эсте (91%); в равных матчапах они в норме, блёнкет-нёрф испортил
бы их ради ботового артефакта.

**Исправленные баги (логика/UX):**
- **BUG-B (UX):** после «Новая партия» второй игрок застревал на экране итога —
